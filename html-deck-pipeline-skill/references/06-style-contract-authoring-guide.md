# Style-Contract 编写指南

> 本文档是"元指南"——定义什么构成一个好的 style-contract，以及如何从零创建一个新风格。

## 1. 什么是 Style-Contract

Style-contract 是一份 Markdown 文档，为特定视觉风格定义所有设计令牌和规则。它是 sub-agent 生成 HTML 时的**唯一视觉权威**，确保多分片、多轮迭代下风格一致。

文件命名：`style-contract-{style-id}.md`

## 2. 最低内容基线

一个合格的 style-contract 必须包含以下部分：

| # | 必须部分 | 说明 |
|---|---------|------|
| 1 | **风格标识** | `style-id`、风格名称、一句话气质描述 |
| 2 | **色彩令牌** | 至少 12 个 CSS 自定义属性，覆盖背景/文字/强调/功能四大类 |
| 3 | **字体栈** | `--font-main` + `--font-code`，含中文回退 |
| 4 | **容器规格** | 舞台比例策略（默认 16:9，可选 4:3 / 16:10 / adaptive）、最大宽度、内边距 |
| 5 | **页面模式定义** | 至少覆盖 7 页模板标准中的全部页面类型 |
| 6 | **视觉增强工具箱** | 列出该风格使用的增强手段，至少覆盖 4 个类别 |
| 7 | **禁止事项** | 明确列出该风格不应出现的视觉元素或文案模式 |

## 3. 色彩令牌系统

### 3.1 命名规范

使用 `--` 前缀的 CSS 自定义属性，采用语义化命名（描述用途而非颜色值）：

```
--bg          主背景色
--bg-d        次级/较深背景色
--bg-card     卡片/面板背景色
--text        主文本色
--text-sec    次级文本色
--accent      主强调色
--accent-d    强调色深色变体
--brand       品牌色（可与 accent 相同）
--ok          正面/成功色
--warn        警告/风险色
--tip-bg      提示框背景色
--tip-bd      提示框边框色
--shadow      阴影色
--glass-bg    毛玻璃背景色
--glass-bd    毛玻璃边框色
--status      底部状态信息颜色
--code-bg     代码块背景色
```

### 3.2 最低数量

- **基础类**（必须 ≥ 5）：`--bg`, `--text`, `--accent`, `--bg-card`, `--text-sec`
- **功能类**（必须 ≥ 4）：`--ok`, `--warn`, `--tip-bg`, `--tip-bd`
- **质感类**（推荐 ≥ 3）：`--shadow`, `--glass-bg`, `--status`

### 3.3 对比度验证

所有令牌组合必须满足 WCAG 2.2 AA：
- `--text` 对 `--bg` ≥ 4.5:1
- `--text` 对 `--bg-card` ≥ 4.5:1
- `--text-sec` 对 `--bg` ≥ 4.5:1（若用于正文大小文本）
- `--accent` 对 `--bg` ≥ 3:1（作为非文本控件色）

## 4. 字体栈规范

标准格式（中文优先回退链）：

```css
--font-main: "Segoe UI", "PingFang SC", "Microsoft YaHei", "Noto Sans SC", sans-serif;
--font-code: "Cascadia Code", "Fira Code", "Source Code Pro", "Consolas", monospace;
```

要求：
- 必须包含至少一个中文字体（`PingFang SC` / `Microsoft YaHei` / `Noto Sans SC`）
- 必须以通用族名（`sans-serif` / `monospace`）收尾
- `--font-code` 推荐支持连字的等宽字体

## 5. 视觉增强工具箱

六大类别，合格的 style-contract 须覆盖**至少 4 类**：

| 类别 | 典型手段 | 适用场景 |
|------|---------|---------|
| **背景层** | 径向渐变、纹理叠加、网格底纹 | 封面页、章节分隔 |
| **信息层** | 高亮标记、标签/徽章、语义 emoji、结构符号 | 关键概念强调、步骤引导 |
| **结构层** | 分隔线、栅格布局、时间轴、卡片分组 | 多元素页面组织 |
| **动效层** | 入场渐现、hover 缩放、翻页过渡 | 页面切换、交互反馈 |
| **质感层** | 阴影层次、毛玻璃效果、圆角梯度 | 卡片层次、面板区分 |
| **强调层** | 品牌色贯穿、图标重复母题、装饰性符号 | 品牌识别、视觉节奏 |

每个类别无需穷举所有手段，但须至少列出该风格实际使用的 1-3 个具体手段及参数。

## 6. 从真实产品提取设计语言（5 步法）

当需要基于真实网站或产品创建新的 style-contract 时，按以下步骤进行：

### Step 1：信息架构骨架

分析目标产品的页面结构，识别核心区块及其排列顺序。

**示例输出**：`Hero → Features（卖点卡） → How（三步流程） → Demos（演示区） → CTA（行动号召） → Footer`

关注点：
- 区块数量与密度
- 信息递进关系（认知→兴趣→行动）
- 留白节奏

### Step 2：视觉语言提取

从产品截图中采样关键视觉参数：

- **气质关键词**：3-5 个形容词（如"专业克制""活力亲和""深沉科技"）
- **版式节奏**：大标题-正文-留白的比例感
- **色彩采样**：使用取色器提取主色、辅色、强调色、背景色（记录 HEX 值）
- **圆角/阴影参数**：`border-radius` 和 `box-shadow` 的具体数值

### Step 3：组件风格映射

将产品中的 UI 组件映射到讲稿模板的 7 页类型：

| 产品组件 | 映射到讲稿页面类型 |
|---------|-------------------|
| Hero Banner | 封面页 |
| Feature Cards | 方法/流程页 |
| Comparison Table | 对比页 |
| Testimonials | 证据页 |
| Step-by-step Guide | 流程/命令页 |
| CTA Section | 行动页 |
| Footer | 结尾页 |

### Step 4：文案语气模型

分析产品文案的语言风格，提炼为可复制的规则：

- **句式特征**：短句/长句、主动语态/被动语态
- **动词密度**：高动词（行动导向）vs 形容词（描述导向）
- **人称视角**：第一人称（我们）、第二人称（你）、第三人称
- **情感基调**：理性克制 / 热情激励 / 轻松幽默

### Step 5：复刻检查清单

完成 style-contract 编写后，逐项验证：

- [ ] 色彩令牌数量 ≥ 12
- [ ] 字体栈含中文回退
- [ ] 7 页模板类型均有视觉定义
- [ ] 视觉增强工具箱覆盖 ≥ 4 类
- [ ] 气质关键词与产品一致
- [ ] 对比度满足 WCAG 2.2 AA
- [ ] 禁止事项已列出
- [ ] 用 showcase HTML 验证视觉效果

## 7. 合格范例

当前技能包含三套通过审核的 style-contract，可作为编写参考：

| style-id | 风格名称 | 特点 |
|----------|---------|------|
| `dark-theme` | 深蓝黑沉浸版 | 深色背景 + 蓝色强调 + 毛玻璃质感 |
| `light-theme` | 清透白蓝版 | 浅色背景 + 蓝色系 + 简洁干净 |
| `qclaw-theme` | 深色红轴版 | 深灰黑背景 + 红色品牌强调 + 本地资源引用 |

参见 `examples/{style-id}/style-contract-{style-id}.md` 获取完整示例。
