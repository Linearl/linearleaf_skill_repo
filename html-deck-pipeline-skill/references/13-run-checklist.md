# 执行清单（Run Checklist）

用于运行阶段的勾选，不与主流程正文耦合。

## 通用

- [ ] 如需网页资料抓取，优先内嵌 `scrapling-web-fetch`
- [ ] 每阶段交互记录均已落盘到 `{work_dir}/00-input/comms/`
- [ ] 交互记录文件名均带阶段字母前缀（`A-` / `B-` / `C-` / `D-` / `E-` / `F-`）

## 阶段 A

> 完整问询模板见 `04-stage-a-question-card.md`。

- [ ] 完成首轮 `ask_questions`
- [ ] 完成结构化问询（含工作目录）
- [ ] 完成屏幕分辨率 / 缩放 / 投屏可视区问询
- [ ] 完成 `ratio_mode` 选择并冻结（默认 `16:9`，可选 `4:3` / `16:10` / `adaptive`）
- [ ] 已基于屏幕环境回写 3 条适配建议
- [ ] 完成每轮“总结 + 用户确认”
- [ ] 完成未明确项判定与追问闭环
- [ ] 获得“可进入阶段 B”确认
- [ ] 完成初始化与记录落盘
- [ ] 输出冻结快照

## 阶段 B

- [ ] `B-framework-v-XX.md` 已生成并就绪
- [ ] 冻结总分总结构
- [ ] 产出拆分方案与页数估算
- [ ] 在 `ask_questions` 之前先输出页码级大纲（页码/分片/页面类型/核心信息）
- [ ] 用 `ask_questions` 完成用户确认
- [ ] 不认可时完成重拆并再确认
- [ ] 阶段收口时用 `ask_questions` 获得“可进入阶段 C”确认

## 阶段 C

- [ ] 分镜稿完成并编号
- [ ] 风格资产继承正确
- [ ] 已在分镜中显式记录 `style-contract` 与 `style-showcase` 路径
- [ ] Emoji/符号美化已在分镜阶段完成并冻结（不延后到 HTML 阶段）
- [ ] Emoji 风格参考已对齐 `11-组内分享/docs/3.其他/v1`（风格参考，不复制内容）
- [ ] 流程步骤优先使用语义 emoji（如 `🎯/🔍/🛠️/🧪/✅`），不强制使用 emoji 数字序号（`1️⃣/2️⃣/3️⃣`）
- [ ] 分镜已标注页面模式类型与背景呈现策略
- [ ] 分镜已细化到界面元素布局层（表格/卡片/流程/提示等）
- [ ] 方法页/流程页/对比页均含至少 1 个具体示例或模板片段
- [ ] 已由 AI 生成 `source-coverage-strike.md`（已覆盖内容=删除线；未删除=潜在省略）
- [ ] AI 已完成覆盖率自评估（覆盖率估值 / 高优先级遗漏 / 补充建议）
- [ ] 已通过 `ask_questions` 完成人工审核确认（遗漏判断与补充决策）
- [ ] 覆盖率达到门禁阈值（默认 ≥70%），或已记录豁免理由
- [ ] 分镜门禁通过
- [ ] 阶段收口时用 `ask_questions` 获得“可进入阶段 D”确认

## 阶段 D

- [ ] 进入 D.1 前已读取 `references/17-style-namespacing-rules.md`
- [ ] HTML 分片生成完成
- [ ] D.2 自查发现需修改时，已通过 `ask_questions` 确认修改范围再动手
- [ ] 并行前提示已给到 sub-agent
- [ ] sub-agent 已回传“已读取 style-contract + style-showcase”清单
- [ ] 样式美观、信息密度、布局多样性通过分片自查
- [ ] 背景样式类型不超过 3 种且章节内保持一致
- [ ] 已提示用户查看分片 HTML，并收集修改意见
- [ ] D.3 获批后，若分片 HTML 有修改，已一次性批量回写对应分镜稿（同版本、同分片编号）
- [ ] 阶段收口时用 `ask_questions` 获得“可进入阶段 E（合并与发布校验）”确认

## 阶段 E

- [ ] 合并前样式预检完成（类冲突/命名空间/表格对齐）
- [ ] 合并与页序校验通过
- [ ] HTML 门禁通过
- [ ] 合并稿 `<title>` 符合 `{主题}-合并(v-XX)`
- [ ] 信息密度与标杆对齐通过 `07` §2.10 全部 5 项门禁
- [ ] 样式多样性通过 `07` §2.0 门禁（详细规则见 `10`）
- [ ] 确认不存在 `#progress` 或同义进度条元素
- [ ] 确认不存在 `prevBtn` / `nextBtn`
- [ ] 已完成 preview 验收并覆盖正式合并稿（含二次门禁）
- [ ] 阶段收口时用 `ask_questions` 获得“本轮收口 / 进入阶段 F（归档总结）”确认

## 阶段 F

- [ ] 已完成版本资产归档（分镜/HTML/测试报告/关键沟通记录）
- [ ] 已完成经验教训总结（可复用做法/失误模式/规避建议）
- [ ] 已用 `ask_questions` 询问是否将经验教训回传到技能目录
