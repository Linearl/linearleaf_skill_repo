# 执行清单（Run Checklist）

## 通用

- [ ] 如需网页资料抓取，优先内嵌 `scrapling-web-fetch`
- [ ] 每阶段交互记录均已落盘到 `{work_dir}/00-input/comms/`
- [ ] 交互记录文件名均带阶段字母前缀（`A-` / `B-` / `C-` / `D-` / `E-` / `F-`）

## 阶段 A

- [ ] 完成首轮 `ask_questions`
- [ ] 完成结构化问询（含工作目录）
- [ ] 完成屏幕分辨率 / 缩放 / 投屏可视区问询
- [ ] 完成 `ratio_mode` 选择并冻结（默认 `16:9`，可选 `4:3` / `16:10` / `adaptive`）
- [ ] 已基于屏幕环境回写 3 条适配建议
- [ ] 完成每轮“总结 + 用户确认”
- [ ] 完成未明确项判定与追问闭环
- [ ] 获得“可进入阶段 B”确认
- [ ] 完成初始化与记录落盘
- [ ] 输出冻结快照

## 阶段 B

- [ ] `B-framework-v-XX.md` 已生成并就绪
- [ ] 冻结总分总结构
- [ ] 产出拆分方案与页数估算
- [ ] 在 `ask_questions` 之前先输出页码级大纲（页码/分片/页面类型/核心信息）
- [ ] 用 `ask_questions` 完成用户确认
- [ ] 不认可时完成重拆并再确认
- [ ] 阶段收口时用 `ask_questions` 获得“可进入阶段 C”确认

## 阶段 C

- [ ] 分镜稿完成并编号
- [ ] 风格资产继承正确
- [ ] 已在分镜中显式记录 `style-contract` 与 `style-showcase` 路径
- [ ] Emoji/符号美化已在分镜阶段完成并冻结（不延后到 HTML 阶段）
- [ ] Emoji 风格参考已对齐 `11-组内分享/docs/3.其他/v1`（风格参考，不复制内容）
- [ ] 流程步骤优先使用语义 emoji（如 `🎯/🔍/🛠️/🧪/✅`），不强制使用 emoji 数字序号（`1️⃣/2️⃣/3️⃣`）
- [ ] 分镜已标注页面模式类型与背景呈现策略
- [ ] 分镜已细化到界面元素布局层（表格/卡片/流程/提示等）
- [ ] 方法页/流程页/对比页均含至少 1 个具体示例或模板片段
- [ ] 已由 AI 生成 `source-coverage-strike.md`（已覆盖内容=删除线；未删除=潜在省略）
- [ ] AI 已完成覆盖率自评估（覆盖率估值 / 高优先级遗漏 / 补充建议）
- [ ] 已通过 `ask_questions` 完成人工审核确认（遗漏判断与补充决策）
- [ ] 覆盖率达到门禁阈值（默认 ≥70%），或已记录豁免理由
- [ ] 分镜门禁通过
- [ ] 阶段收口时用 `ask_questions` 获得“可进入阶段 D”确认

## 阶段 D

- [ ] 进入 D.1 前已读取 `references/19-website-skeleton-spec.md` §8（slide 模板格式）
- [ ] HTML 页面已按 `slides/<part_id>/<NN-description>.html` 生成
- [ ] `slides-config.json` 随页面生成同步维护
- [ ] D.2 已运行 `python container/serve.py <target_dir> --theme <theme_name>` 启动服务
- [ ] 网站骨架验证通过：翻页、章节导航、hash 路由、自适应缩放
- [ ] 导出 HTML 和导出 PPTX 按钮可见且可点击
- [ ] 主题切换功能正常
- [ ] D.3 已运行 `python scripts/measure_utilization.py http://localhost:8080` 量化空白区域
- [ ] D.3 自查发现需修改时，已通过 `ask_questions` 确认修改范围再动手
- [ ] 样式美观、信息密度、布局多样性通过自查
- [ ] D.4 已提示用户在浏览器中查看完整 deck，并收集修改意见
- [ ] D.4 如需章节额外样式，已按 `.part-<part_id>` 命名空间放入 `v-XX/style/<part_id>.css`
- [ ] D.4 获批后，若 HTML 有修改，已一次性批量回写对应分镜稿
- [ ] 阶段收口时用 `ask_questions` 获得”可进入阶段 E”确认

## 阶段 E

- [ ] E.1 已同时导出 HTML 和 PPTX，文件落盘到 `20-html/v-XX/`
- [ ] 导出文件已实际打开验证（翻页、样式、无 404）
- [ ] E.1 已完成定版后回灌分镜稿（HTML 文案 → `10-storyboards/v-XX/`）
- [ ] 用户验收通过，正式版本已冻结
- [ ] 如需版本升级，已复制 `20-html/v-{旧}/` 和 `10-storyboards/v-{旧}/` 到新版本目录
- [ ] 阶段收口时用 `ask_questions` 获得”本轮收口 / 进入阶段 F”确认

## 阶段 F

- [ ] 已完成版本资产归档（分镜/HTML/测试报告/关键沟通记录）
- [ ] 已完成经验教训总结（可复用做法/失误模式/规避建议）
- [ ] 已用 `ask_questions` 询问是否将经验教训回传到技能目录
