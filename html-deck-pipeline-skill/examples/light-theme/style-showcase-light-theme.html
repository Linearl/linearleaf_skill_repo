◇ 封面模板 / Cover P1/7
✧ 总分总：开场总述

组内分享通用封面模板

v1 主题与版式更强调“先交代路径，再进入分论点”，适合稳定交付、节奏明确的分享型 PPT。

场景 分享

适合组会、方法论汇报、经验复盘。

结构 总分总

先总述，再拆解，最后回收行动。

节奏 稳定

便于多人协作和版本复用。

▣ 封面页建议

  • 主标题、副标题、分享对象、日期。
  • 一句话说明这次分享要解决什么问题。
  • 可附“结构提示”，帮助观众预期后续节奏。
推荐用途：总稿封面 / 章节开头 Light Theme 示例
◇ 导航模板 / Route P2/7
✧ 导航页

这一讲怎么展开：开头、拆解、收口

导航页是 v1 的稳定器，先告诉观众“接下来按什么顺序讲”，再开始展开细节。

① 开头总述

统一问题背景、目标和分享边界。

② 方法拆解

讲清楚做事步骤、判断标准和使用条件。

③ 证据与对比

用表格、对比、案例证明方法为什么有效。

④ 行动收口

明确可复制动作和下次复盘节点。

▣ 导航页必备元素

  • 章节顺序或阶段名称。
  • 本次重点与不展开内容。
  • 如有听众预期差，可提前说明阅读方式。

✧ 可选元素

可附上时间分配、提问位置、资料附录入口等信息。

ℹ 提示框：建议按 P2 → P3 → P4 的顺序先看路径、再看方法、最后看证据。
模板类型：导航页 先讲路径，再讲内容
◇ 方法模板 / Method P3/7
✧ 方法页

方法拆解：从问题定义到可执行结论

方法页适合用“结论卡 + 步骤流 + 边界提示”来表达，不追求炫技，追求听众一眼知道怎么照着做。

STEP ① 定义问题

明确对象、范围、输入材料，避免方法还没开始就跑偏。

STEP ② 统一标准

提前写清“好 / 不好”的判断标准，减少事后解释成本。

STEP ③ 生成动作

把结论落成下一步动作，并给出验证信号。

▣ 方法页必备元素

  • 步骤编号、输入、动作、输出。
  • 边界条件或不适用场景。
  • 一句话提醒：听众看完该记住什么。

✧ 可选强化

可补充适用角色、参考模板链接、失败案例提示。

⚠ 失败分支提示：若步骤 2 标准不一致，先统一口径再继续步骤 3。
模板类型：方法拆解页 适合“怎么做”类内容
◇ 证据矩阵模板 / Matrix P4/7
✧ 证据矩阵页

证据页：用简单表格承载结论、依据与例外

v1 不要求每页都“设计感爆棚”，但要求结构稳定，所以证据页推荐矩阵或表格版式，方便复用与核对。

观察项 结论 证据 例外提醒
输入质量 决定输出上限 样本完整度、更新频率 急稿场景可降级处理
方法一致性 决定团队可复用性 模板约束、命名规范 创新页可适度破例
行动闭环 决定分享是否真正有用 责任人、时间点、验收标准 仅启发型分享可弱化

▣ 矩阵页必备元素

  • 表头明确，行列语义稳定。
  • 至少一列专门承载“例外/风险”。
  • 表格适合静态信息，便于读者抄走结论。
模板类型：证据矩阵页 适合稳定、可复核的信息展示
◇ 对比模板 / Compare P5/7
✧ 对比页

推荐写法 vs 常见问题：同一主题如何讲得更清楚

这类页面适合做规范培训：左边写推荐做法，右边写常见失误，让团队更容易形成一致风格。

✅ 推荐写法（Do）
  • 标题直接写结论或用途。
  • 每页只讲一个中心问题。
  • 把行动放在页面收口位置。
⚠ 常见问题（Don't）
  • 标题空泛，如“情况说明”“一些思考”。
  • 一页混入太多主题，听众只能礼貌点头。
  • 没有行动收口，分享结束后无人知道要做什么。
🧭 讲解提示（How）
  • 先讲左栏，再指出右栏问题，最后总结迁移规则。
  • 不要把两栏写成两段长文，每栏 3 条内最好。
模板类型：对比页 适合规范培训、经验复盘、A/B 方案对照
◇ 行动模板 / Action P6/7
✧ 行动页

行动清单：谁来做、何时做、怎么判断做完

v1 的行动页建议直接按职责拆卡，避免漂亮但无执行价值的“大团圆结尾”。

👤 负责人
  • 明确主责人与协同方。
  • 跨部门动作单独标注。
🗓 时间点
  • 写截止日期或检查频率。
  • 能周更就不要写“尽快”。
✅ 验收标准
  • 最好有可观测输出或文档产物。
  • 避免“感觉差不多了”式验收。

▣ 行动页必备元素

责任、时间、标准三件套缺一不可，否则行动页会退化成情绪表达页。

✧ 可选元素

可增加风险预案、优先级标签、依赖资源和会后同步方式。

模板类型：行动页 用执行闭环结束“分”的部分
◇ 底页模板 / Closing P7/7
✧ 底页

收口页：总结一句话，留下一条会后动作

底页延续“总分总”里的最后一个“总”，回收前文要点，并给听众一个干净的退出点。

◆ 一句话总结

好的分享不是把材料讲完，而是让听众知道下一步怎么做。

📌 会后动作

同步材料、确认负责人、约定下一次回看节点。

✧ 底页可选元素

  • 联系方式、团队名称、参考资料入口。
  • 附录提示、FAQ、更新日期。
  • 如需更正式，可添加版权或使用范围说明。
模板类型：底页 / 收口页 封面与底页构成完整前后呼应
