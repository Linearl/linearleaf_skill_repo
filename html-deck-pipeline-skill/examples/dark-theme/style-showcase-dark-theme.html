◉ 目录模板 / Navigation P2/7
✦ 目录页

阅读路径与章节分工

目录页负责告诉观众“从哪里开始、经过什么、最后落到哪里”，避免一上来丢结论像扔飞盘。

① 背景校准
  • • 讲清观察区间与触发信号
  • • 先校准问题再给方案
② 证据拆解
  • • 数据/访谈/来源交叉验证
  • • 明确成立条件与边界
③ 策略修正
  • • 拆解停/调/加三类动作
  • • 每条动作对应验收信号
④ 执行收口
  • • 确认责任人与检查节奏
  • • 同步补齐风险与回退护栏

▣ 目录页必备信息

  • 4 段式路径或同级章节分组。
  • 本段目标与建议阅读时长。
  • 若有风险章节，提前点出“高风险页”位置。

✧ 可选强化

可附上“本次不展开内容”或“先看结论请跳到 P6”提示。

⚠ 提示框：先看 P3 的方案对比，再看 P4 的证据链，最后在 P6 决定执行动作。
模板类型：导航页 / 目录页 路径先行，降低认知切换成本
◉ 双栏对比模板 / Compare P3/7
✦ 对比页

旧判断 vs 新判断：问题出在假设还是执行？

对比页适合放两个版本、两个方案、两个阶段的差异，帮助观众迅速看见“为什么要改”。

原方案：等库存自然去化
  • 默认需求修复速度快于供给反应速度。
  • 执行重点在“观察”，行动门槛设置偏高。
  • 适合低噪音环境，但对突发消息敏感度不足。
新方案：把执行风险前置
  • 把跟踪重点从“总量变化”切到“兑现节奏变化”。
  • 新增预警阈值、复盘频率和责任人。
  • 风险点显式写出，避免只靠颜色暗示。

▣ 适合内容

旧版/新版、国内/海外、乐观/审慎、A/B 路径。

▣ 表达要点

标题直接点出“比较对象 + 决策含义”，别写成“情况对比”这种空气标题。

✧ 可选元素

可加变化箭头、阶段时间轴、胜负手标签。

模板类型：双栏对比页 适合讲“为什么改变”
◉ 数据证据模板 / Evidence P4/7
✦ 数据证据页

关键证据页：指标、来源、口径要同时出现

Scheme C 的证据页不是只摆数字，而是把数据、来源、时间截面、结论方向一起钉在页面上。

库存周转 11.2 天

较上期缩短 1.8 天

现货贴水 -2.4%

连续三周收敛

执行风险 中高

集中在消息兑现节奏

▣ 来源与口径

来源 A｜行业周报

观察窗口：最近 8 周；口径：样本仓库库存。

来源 B｜现货访谈

观察窗口：近 2 周；口径：核心区域成交反馈。

来源 C｜价格数据库

观察窗口：日频；口径：主力合约收盘价。

◆ 一句话结论

证据指向“方向没变，但兑现顺序已变”，所以本轮修正重点不是改方向，而是改执行节奏。

模板类型：数据证据页 数字、来源、结论三者同屏
◉ 流程模板 / Process P5/7
✦ 流程页

从监测到调整：策略修正的 3 步流

流程页适合说明执行顺序、责任节点与回滚条件，避免策略建议写成口号墙。

STEP ① 发现偏差

监测库存、贴水、成交结构三个信号，任一项触发阈值即进入复核。

STEP ② 交叉验证

用数据源、访谈源、价格源做三角校验，确认是噪音还是结构变化。

STEP ③ 执行修正

更新阈值、责任人、节奏表，并记录下次复盘时间点。

▣ 流程页必备元素

  • 步骤编号或阶段名称。
  • 每步的输入、动作、输出。
  • 异常情况下的回滚或加密监控点。

✧ 可选强化

可加入责任角色、时间刻度、升级路径或外部依赖提醒。

⚠ 失败分支提示：若 STEP ② 校验冲突，回退到 STEP ① 重新核对输入口径。
模板类型：流程页 适合讲执行顺序与回滚点
◉ 行动模板 / Action P6/7
✦ 行动页

从判断落到动作：触发条件、关键动作、验收信号

行动页适合放“下一步怎么办”，是从分析稿变成交付稿的最后一脚。

模块编号培训要点可执行动作

基础篇2.2把问题解决在早期编码前输出候选方案并写明取舍
进阶篇2.6并行加速与工作集正交同文件同一时刻仅一个执行者修改

模板类型：行动页 分析结束，执行开始
◉ 底页模板 / Closing P7/7
✦ 底页

结论回收与会后动作留口

底页不是“谢谢观看”四个字孤零零地站着，而是用来收结论、留动作、挂附录入口。

◆ 一句话回收

风险没有消失，只是从“是否成立”转成“何时兑现”。

📌 会后动作

补齐附录、同步责任人、确认下一次复盘时间。

📎 附录入口

可在此挂“口径说明、来源清单、页码映射、FAQ”。

✧ 底页可选元素

  • 联系人 / 团队名称 / 更新时间。
  • 附录索引、二维码、后续议程。
  • 如需保守表达，可放“供内部讨论使用”。
模板类型：底页 / 收口页 封面与底页已完整闭环
