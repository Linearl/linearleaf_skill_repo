> 从零到一部署 Scrapling 爬虫框架的完整经验文档
> 
> 验证环境：Windows 11, Python 3.12.7 (Conda), Scrapling 0.4.2
> 
> 更新时间：2026-03-11

---

## 一、什么是 Scrapling？

Scrapling 是一个 **自适应 Web 抓取框架**（GitHub: [D4Vinci/Scrapling](https://github.com/D4Vinci/Scrapling)，28k+ stars），核心能力：

| 能力 | 说明 |
|---|---|
| **StealthyFetcher** | 绕过 Cloudflare Turnstile 等反爬系统 |
| **自适应解析** | 网站改版后自动重新定位目标元素 |
| **Spider 爬虫** | Scrapy 风格，支持断点续爬、流式输出 |
| **MCP Server** | 与 Claude/Cursor/VS Code Copilot 集成 |
| **CLI 工具** | 命令行直接抓取，无需写代码 |

它是 Crawl4AI 的互补工具——当 Crawl4AI 因反爬抓不到数据时，用 Scrapling 的 StealthyFetcher 先获取页面。

## 二、安装步骤（5 分钟）

### 前提条件
- **Python 3.10+**（推荐 3.12）
- **Conda**（推荐）或 venv

### Step 1: 创建/激活 Python 环境

```bash
# 若使用 Conda（推荐）
conda create -n scrapling python=3.12 -y
conda activate scrapling

# 若使用 venv
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate
```

### Step 2: 安装 Scrapling 全功能版

```bash
pip install "scrapling[all]"
```

各子模块说明：
- `scrapling[fetchers]` — Fetcher + StealthyFetcher + DynamicFetcher
- `scrapling[ai]` — MCP Server 支持
- `scrapling[shell]` — 交互式 Shell + CLI extract 命令
- `scrapling[all]` — 以上全部

### Step 3: 安装浏览器依赖

```bash
scrapling install
```

这一步会安装 Playwright Chromium 和指纹伪装组件。若失败可尝试：
```bash
scrapling install --force
```

### Step 4: 验证安装

```bash
python -c "import scrapling; print(scrapling.__version__)"
# 预期输出: 0.4.2
```

## 三、踩坑记录

### 坑 1：`scrapling` 命令不在 PATH 中

**现象：**
```
scrapling : 无法将"scrapling"项识别为 cmdlet、函数、脚本文件或可运行程序的名称
```

**原因：** `scrapling` 安装到了 Python 环境的 `Scripts` 目录中，但该目录不在系统 PATH 中。

**解决方案（任选一）：**

```bash
# 方案 A：使用完整路径（替换为你实际的 Python 环境路径）
C:/Users/<用户名>/anaconda3/Scripts/scrapling install
C:/Users/<用户名>/anaconda3/Scripts/scrapling extract get "https://example.com" out.md

# 方案 B：激活 Conda 环境后使用
conda activate scrapling    # 或你的环境名
scrapling install

# 方案 C：添加 Scripts 目录到 PATH
$env:PATH += ";C:\Users\<用户名>\anaconda3\Scripts"
```

### 坑 2：`python -m scrapling` 不可用

**现象：** `No module named scrapling.__main__`

**原因：** Scrapling 没有 `__main__.py`，不支持 `python -m scrapling` 方式运行。

**解决：** 只能用 `scrapling` CLI 命令（参见坑 1 的解决方案）。

### 坑 3：`scrapling install` 时出现 Node.js DeprecationWarning

**现象：**
```
(node:1360) [DEP0169] DeprecationWarning: `url.parse()` behavior is not standardized...
```

**处理：** 这是 Node.js/Playwright 内部的警告，**不影响功能**，可以安全忽略。

### 坑 4：网络问题导致浏览器下载失败

**现象：** `scrapling install` 超时或下载中断。

**解决方案：**
```bash
# 方案 A：设置代理
set HTTPS_PROXY=http://127.0.0.1:7890
scrapling install

# 方案 B：使用 Docker（浏览器已预装）
docker pull pyd4vinci/scrapling
```

### 验证 2：Python API Fetcher（已验证通过 ✅）

```python
from scrapling.fetchers import Fetcher

page = Fetcher.get('https://quotes.toscrape.com/')
print(f'Status: {page.status}')  # 200

quotes = page.css('.quote .text::text').getall()
print(f'Quotes found: {len(quotes)}')  # 10

```

### 验证 3：StealthyFetcher 反反爬

```python

    'https://nopecha.com/demo/cloudflare',
    headless=True

```

### 验证 4：MCP Server

```bash
scrapling mcp
# 应启动 MCP Server，监听 stdio 输入
# Ctrl+C 退出
```

### 验证 5：Spider 爬虫框架

```python
from scrapling.spiders import Spider, Response

class QuotesSpider(Spider):
    name = "quotes"
    start_urls = ["https://quotes.toscrape.com/"]
    concurrent_requests = 5

    async def parse(self, response: Response):
        for quote in response.css('.quote'):
            yield {
                "text": quote.css('.text::text').get(),
                "author": quote.css('.author::text').get(),
            }
        next_page = response.css('.next a')
        if next_page:
            yield response.follow(next_page[0].attrib['href'])

result = QuotesSpider().start()
print(f"共抓取 {len(result.items)} 条数据")
result.items.to_json("quotes.json")
```

## 五、VS Code MCP 集成配置

在 `.vscode/settings.json` 中添加：

```json
{
    "chat.tools.autoApprove": "allowed",
    "mcp": {
        "servers": {
            "scrapling": {
                "command": "scrapling",
                "args": ["mcp"],
                "env": {}
            }
        }
    }
}
```

> **注意：** 如果 `scrapling` 不在 PATH 中，`command` 需要写完整路径，例如：
> `"command": "C:/Users/<用户名>/anaconda3/Scripts/scrapling"`

配置完成后，在 VS Code Copilot Chat 中即可通过 MCP 调用 Scrapling 功能。

## 六、常用命令速查

```bash
# CLI 抓取为 Markdown
scrapling extract get 'https://example.com' output.md

# CLI 隐身抓取（绕过 Cloudflare）
scrapling extract stealthy-fetch 'https://target.com' output.html --solve-cloudflare

# CLI 指定 CSS 选择器抓取
scrapling extract get 'https://example.com' output.md --css-selector '.article-content'

# 交互式 Shell
scrapling shell

# 启动 MCP Server
scrapling mcp

# 重装浏览器依赖
scrapling install --force
```
