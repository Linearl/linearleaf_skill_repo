# HTML 演示文稿生成工作流（防止上下文循环总结）

## 背景

当主文档过长、需求频繁变更时，直接在单上下文里迭代易出现：
- 不断总结而不落盘
- 改动范围漂移
- 页序与分镜不一致

本流程通过“拆分 + 并行 + 脚本校验 + 单点收敛”降低失控风险。

## 标准步骤

### 0) 专题目录初始化（新增）

- 先创建专题目录并固定四个子目录：
  - `00-input`（输入原稿副本）
  - `10-storyboards`（分镜稿）
  - `20-html`（分片和合并 HTML）
  - `90-tests`（测试记录）
- 推荐使用脚本：`scripts/init_topic_folder.py`
- 文件编号规则见：`references/numbering-rules.md`

### 1) 主文档拆分

- 按自然章节边界拆分（如 2.1~2.4 / 2.5~2.8 / 2.9~2.11+收口）。
- 每段定义清晰输入输出：
  - 输入：对应章节正文
  - 输出：分镜稿 + 对应 HTML 分片

### 2) 分镜并行产出

- 使用 sub-agent 并行处理互不依赖的段落。
- 每段采用固定模板：
  - 第X页标题
  - 页面目标
  - 页面完整文案
  - 演讲备注（含过渡句）

### 3) 单段落地与单段校验

- 每个分镜段先落地单独 HTML。
- 本段内部先校验页序、标题、动作页是否完整。

### 4) 合并构建

- 统一运行项目脚本：`组内分享/merge_scheme_c_parts.py`。
- 保证合并后：
  - `active` 唯一
  - `data-index` 连续
  - 页码显示与总页数一致

### 5) 回归验证

- 结构验证：页序、关键起始页、收口页。
- 内容验证：禁用词、口播动作、映射附录、质量清单。
- 建议命令（专题目录模式）：
  - `python scripts/validate_storyboards_generic.py --root "组内分享/docs/{专题目录}" --glob "10-storyboards/*分镜稿.md"`

### 6) 改进闭环

- 让 agent 提出改进建议（编号化）。
- 用户批准后再批量执行。
- 执行日志写入“改进执行记录”文档。

## 推荐分工

- **Agent A（分析）**：拆分边界、风险识别、建议清单。
- **Agent B/C（并行内容）**：分镜生成、页面文案增强。
- **主 Agent（收敛）**：落盘、脚本执行、回归、合并输出。

## 成功判据

- 任务从“循环总结”转为“持续产出文件”。
- 每轮都有可见增量（文档或 HTML 变化）。
- 合并稿页序与分镜一致，且可直接预览讲解。
